Noosphere S4 EEG Encoder — Benchmark Report

Generated 2026-04-21 02:55:56  ·  Noosphere v1.6.0  ·  5 datasets  ·  github.com/JosephWoodall/noosphere

Research Summary: This report presents a rigorous evaluation of the Noosphere S4 EEG Encoder using a multi-scale diagonal state-space architecture. We employ a 10-fold Stratified Cross-Validation protocol for within-subject assessment and a Leave-One-Subject-Out (LOSO) protocol for cross-subject generalization. The model integrates complex-valued spectral features (Real+Imaginary), Multi-Head Attention Pooling, and Subject-Invariant Mixup Augmentation to overcome traditional EEG decoding bottlenecks like non-stationarity and phase-insensitivity. Statistical significance is established via two-sided Wilcoxon signed-rank tests against a regularized CSP+LDA baseline, ensuring a robust and mathematically defensible performance profile across heterogeneous neuro-diverse datasets.

S4 outperforms the standard CSP+LDA baseline by +20.6pp within-subject and +15.5pp cross-subject. 4 of 5 datasets reach Wilcoxon significance (p<0.05, paired per-subject). Mean Cohen's κ = 0.379 and AUC-ROC = 0.760.

Methodology

ItemDetail
S4 encoder Regional Multi-Scale S4D: 3 independent cortical heads (Frontal, Motor, Parietal) processing informational channel clusters. Fused into 4× S4D (Diagonal State-Space) blocks (d_model=256). Multi-Head Attention Pooling and Complex Spectral Features. Dirichlet evidential head.
Input Scaling Z-scored per trial; resampled to 256 Hz; 1-second segments (256 samples). Missing channels for low-density arrays are zero-padded to maintain 21-ch input vector.
Reliability Uncertainty-Weighted Accuracy: Performance reported as a function of Dirichlet confidence thresholds, simulating clinical safety gates.
Optimizations torch.compile (reduce-overhead), Automatic Mixed Precision (AMP), and vectorized Spectral GCN.
Hardware Stats OS: Linux 6.19.11-1-cachyos  ·  GPU: NVIDIA GeForce RTX 5070  ·  CPU:
Timing Inference latency measured via torch.cuda.Event for precise kernel-level synchronization (ms/segment).
Pretraining Trunk pretrained on pooled subjects: 150-250 epochs, AdamW, cosine LR (warmup 40 ep), label smoothing ε=0.1, Mixup Augmentation (alpha=0.2), per-class weights, grad clip 1.0
Fine-tuning Head only (trunk frozen): 150 ep max, patience=20 early stop, 15% stratified val split, Mixup (alpha=0.1), best checkpoint restored
Calibration Post-hoc temperature scaling T=1.5 before inference
CSP+LDA baseline OVR CSP (4 filters/class, GEVD), log-variance features, regularised LDA (Ledoit-Wolf λ=0.1). Same splits as S4.
Within-subject 10-fold Stratified Cross-Validation. No leakage.
Statistics Wilcoxon signed-rank (paired per-subject, two-sided). Effect size r = |Z|/√N. Threshold p<0.05.
Noosphere v1.6.0  ·  Datasets via MOABB (Jayaram et al. 2018)  ·  CSP: Blankertz et al. IEEE TBME 2008  ·  ITR: Wolpaw et al. 2000  ·  Wilcoxon 1945

World Model Benchmark — RSSM Sequential Prediction

Protocol: The RSSM world model is trained on frozen RS-S4 embeddings from N−1 subjects (LOSO). At each time step the model observes embedding et, advances its latent state, and imagines the next state via the prior (zero-action baseline). A linear probe on the imagined latent predicts the intent class at trial t+1. Action strategy: zero (no robot/IoT feedback — isolates world model contribution). Architecture: Bare RSSM — GRU deterministic state (256-d) + 16×16 straight-through categorical stochastic latent (DreamerV3 balanced KL).
